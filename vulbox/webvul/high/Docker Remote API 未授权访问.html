漏洞名称 Docker Remote API 未授权访问
详细描述 Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。
docker swarm 是一个将docker集群变成单一虚拟的docker host工具，使用标准的Docker API，能够方便docker集群的管理和扩展，由docker官方提供，它使用Docker Remote API，可以执行docker命令。在使用docker swarm管理docker集群时，被管理的docker节点上会开放一个TCP端口2375，此端口的访问策略是开放的，可以未授权直接访问，所以导致docker信息及系统信息泄露，被攻击者利用后甚至导致获取服务器权限。
解决方案 解决方案：
1、不要2375端口直接暴露在公网；
2、即使端口访问限制在内网中也需要设置严格的访问规则；
3、结合TLS进行访问认证。
官方配置文档：
https://docs.docker.com/swarm/plan-for-production/#cluster-ownership
